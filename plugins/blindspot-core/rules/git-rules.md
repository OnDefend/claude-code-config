# Git Commit Rules

## Commit Messages

- **NEVER include Claude or AI attribution in commit messages** (no Co-Authored-By, no mentions of Claude/AI)
- Keep commit messages concise and descriptive of the changes
- Use conventional commit format when appropriate:
  - `feat:` for new features
  - `fix:` for bug fixes
  - `refactor:` for code refactoring
  - `docs:` for documentation changes
  - `test:` for test additions/changes
  - `chore:` for maintenance tasks

## Branch Naming

- Feature branches: `feature/description` or `feat/description`
- Bug fixes: `fix/issue-number` or `fix/description`
- Refactoring: `refactor/description`

## Pull Requests

- Always create PRs before merging to main/master
- Include a summary of changes
- Reference related issues when applicable
